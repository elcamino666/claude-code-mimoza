# Mimoza Docker Isolation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Containerize Mimoza in Docker so she cannot access the host filesystem, while Ember and Nox continue running natively.

**Architecture:** Build a Docker image with Claude Code (installed via npm) + Bun + the Telegram plugin. The container mounts only Mimoza's Telegram state directory. Auth uses a long-lived token from `claude setup-token`, passed as an env var. A new launcher starts the container via `docker run`, and a simplified watchdog checks container health via `docker inspect`/`docker exec`.

**Tech Stack:** Docker, Node.js 20, Bun, Claude Code (npm), launchd

**Spec:** `docs/superpowers/specs/2026-03-23-mimoza-docker-isolation-design.md`

- [ ] **Step 1.1: Create the Dockerfile**

Create `~/Desktop/claude-code-mimoza/docker/Dockerfile`:

```dockerfile
FROM node:20-slim

# Install dependencies
RUN apt-get update && apt-get install -y \
    git \
    curl \
    unzip \
    procps \
    && rm -rf /var/lib/apt/lists/*

# Install Bun
RUN curl -fsSL https://bun.sh/install | bash
ENV PATH="/root/.bun/bin:$PATH"

# Install Claude Code globally via npm
RUN npm install -g @anthropic-ai/claude-code@2.1.81

# Create working directory (mimoza's sandbox)
RUN mkdir -p /app/workspace && \
    cd /app/workspace && \
    git init && \
    git config user.email "mimoza@bot" && \
    git config user.name "Mimoza"

# Copy CLAUDE.md into the workspace (CLAUDE.md is at repo root = build context)
COPY CLAUDE.md /app/workspace/CLAUDE.md
RUN cd /app/workspace && git add -A && git commit -m "Initial Mimoza setup"

# Copy Telegram plugin
COPY telegram-plugin/ /root/.claude/plugins/cache/claude-plugins-official/telegram/0.0.1/

# Install Telegram plugin dependencies
RUN cd /root/.claude/plugins/cache/claude-plugins-official/telegram/0.0.1 && bun install

# Copy minimal settings
COPY docker/settings.json /root/.claude/settings.json

# Create channels directory (will be mounted from host)
RUN mkdir -p /root/.claude/channels/telegram

# Copy entrypoint
COPY docker/entrypoint.sh /entrypoint.sh
RUN chmod +x /entrypoint.sh

WORKDIR /app/workspace

ENTRYPOINT ["/entrypoint.sh"]
```

### Task 4: Create Docker launcher script

**Files:**
- Create: `~/.local/bin/mimoza-docker-launcher.sh`

- [ ] **Step 4.1: Write the launcher**

Create `~/.local/bin/mimoza-docker-launcher.sh`:

```bash
#!/bin/bash

# ──────────────────────────────────────────────────────────────
# Mimoza Docker Launcher
#
# Runs Mimoza inside a Docker container with filesystem isolation.
# Only the Telegram state directory is mounted from the host.
#
# To see logs:   docker logs -f mimoza
# To restart:    docker restart mimoza
# To shell in:   docker exec -it mimoza bash
# ──────────────────────────────────────────────────────────────

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1"
}

log "Mimoza Docker launcher starting"

# Wait for Docker daemon
RETRIES=0
while ! docker info >/dev/null 2>&1; do
    RETRIES=$((RETRIES + 1))
    if [ "$RETRIES" -gt 30 ]; then
        log "ERROR: Docker daemon not available after 60s, giving up"
        exit 1
    fi
    log "Waiting for Docker daemon... (attempt $RETRIES)"
    sleep 2
done

# Read auth token
TOKEN_FILE="/Users/eni/.claude/channels/telegram-mimoza/.claude-token"
if [ ! -f "$TOKEN_FILE" ]; then
    log "ERROR: Auth token not found at $TOKEN_FILE"
    exit 1
fi
CLAUDE_TOKEN=$(cat "$TOKEN_FILE")

# Stop and remove existing container
docker stop mimoza 2>/dev/null
docker rm mimoza 2>/dev/null

log "Starting Mimoza container"

# Run container
docker run \
    --name mimoza \
    --restart unless-stopped \
    -v /Users/eni/.claude/channels/telegram-mimoza:/root/.claude/channels/telegram \
    -e CLAUDE_CODE_TOKEN="$CLAUDE_TOKEN" \
    -e TELEGRAM_BOT_TOKEN="8613961992:AAFdFLhnhhSXUf8y4kE01kBDKdzOy_LMAC0" \
    mimoza

log "Mimoza container exited"
```

### Task 5: Create Docker watchdog script

**Files:**
- Create: `~/.local/bin/mimoza-docker-watchdog.sh`

- [ ] **Step 5.1: Write the watchdog**

Create `~/.local/bin/mimoza-docker-watchdog.sh`:

```bash
#!/bin/bash
# Mimoza Docker Watchdog — checks container health every 2 minutes

TOKEN="8613961992:AAFdFLhnhhSXUf8y4kE01kBDKdzOy_LMAC0"
CHAT_ID="2042522190"
STATE_FILE="/tmp/mimoza-docker-watchdog-state"

log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1"
}

notify_and_restart() {
    local message="$1"
    log "$message"

    curl -s -o /dev/null "https://api.telegram.org/bot${TOKEN}/sendMessage" \
        -d chat_id="$CHAT_ID" \
        -d text="$message"

    docker restart mimoza
    exit 0
}

log "watchdog check starting"

# Check if Docker is available
if ! docker info >/dev/null 2>&1; then
    log "Docker daemon not running, skipping checks"
    exit 0
fi

# --- Check 1: Container running? ---

CONTAINER_STATUS=$(docker inspect -f '{{.State.Status}}' mimoza 2>/dev/null)
if [ "$CONTAINER_STATUS" != "running" ]; then
    notify_and_restart "Mimoza watchdog: Container not running (status: $CONTAINER_STATUS). Restarting..."
fi

# --- Check 2: Container age (grace period) ---

STARTED_AT=$(docker inspect -f '{{.State.StartedAt}}' mimoza 2>/dev/null)
if [ -n "$STARTED_AT" ]; then
    START_EPOCH=$(date -j -f "%Y-%m-%dT%H:%M:%S" "$(echo "$STARTED_AT" | cut -d. -f1)" "+%s" 2>/dev/null)
    NOW_EPOCH=$(date "+%s")
    if [ -n "$START_EPOCH" ]; then
        ELAPSED=$(( NOW_EPOCH - START_EPOCH ))
        if [ "$ELAPSED" -lt 60 ]; then
            log "container still initializing (${ELAPSED}s old), skipping checks"
            rm -f "$STATE_FILE"
            exit 0
        fi
    fi
fi

# --- Check 3: Claude process alive inside container? ---

CLAUDE_ALIVE=$(docker exec mimoza pgrep -f "claude" 2>/dev/null)
if [ -z "$CLAUDE_ALIVE" ]; then
    notify_and_restart "Mimoza watchdog: Claude process not found inside container. Restarting..."
fi

# --- Check 4: Stuck detection ---

CPU=$(docker stats --no-stream --format '{{.CPUPerc}}' mimoza 2>/dev/null | tr -d '%')

STORED_ZERO=0
if [ -f "$STATE_FILE" ]; then
    STORED_ZERO=$(cat "$STATE_FILE")
fi

if [ "$CPU" = "0.00" ] || [ "$CPU" = "0.0" ]; then
    STORED_ZERO=$(( STORED_ZERO + 1 ))
else
    STORED_ZERO=0
fi

echo "$STORED_ZERO" > "$STATE_FILE"

if [ "$STORED_ZERO" -ge 10 ]; then
    notify_and_restart "Mimoza watchdog: Container appears stuck (no CPU for 20+ min). Restarting..."
fi

log "all checks passed (cpu=${CPU}%, zero_count=$STORED_ZERO)"
```
